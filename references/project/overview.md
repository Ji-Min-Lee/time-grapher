# TimeGrapher — Project Overview

## Background

A **Timegrapher** is a device that analyzes the acoustic signal (beat noise) generated by the escapement vibration of a mechanical watch to diagnose watch performance.
When a watch is placed on a microphone, it analyzes the tick-tock sound and outputs the following measurements in real time.

| Measurement | Description | Normal Range |
|-------------|-------------|--------------|
| Rate (s/d) | Fast/slow deviation per day | Within ±5 s/d recommended |
| Amplitude (°) | Balance wheel angular swing | 270~310° (strong), 220~250° (acceptable) |
| Beat Error (ms) | tick-tock asymmetry | 0.6 ms or less |
| BPH | Beats Per Hour | 21,600 / 28,800 / 36,000, etc. |

## Acoustic Event Structure (Swiss Lever Escapement)

Three acoustic events occur per beat.

```
T1 (A): Impulse pin → Pallet fork    ← most precise → Rate / Beat Error calculation
T2 (B): Escape wheel → Pallet stone  ← irregular    → not used for measurement
T3 (C): Escape wheel locks + Fork → Banking pin  ← with T1 → Amplitude calculation
```

## Project Goals

Extend the baseline `TimeGrapher_v10.5` GUI to design and implement a real-time diagnostic and visualization system running on **Raspberry Pi 5**.

- **Signal Detection** — precise beat event (T1, T3) detection
- **Signal Processing** — real-time filtering (Low-pass / High-pass)
- **Real-Time Analysis** — Rate, Amplitude, Beat Error calculation
- **Real-Time Graphing** — implementation of various display modes

Key architectural objective:
> Design a modular, extensible, and maintainable structure that allows new analyses and graphs to be added without major modifications to existing code.

## System Architecture

```
[Watch]
  ↓ vibration (acoustic signal)
[USB Sensor Stand + Converter Box]
  ↓ USB (analog → digital)
[Raspberry Pi 5]
  ↓ Qt GUI (based on TimeGrapher_v10.5)
[8" Touchscreen (1280×800)]

[Development environment: macOS/Windows PC]  ← Qt Creator + remote development/testing
[Reference device: WeiShi 1000 Standalone Timegrapher] ← measurement value comparison reference
```

## Quality Attributes

| QA | Target | Measurement Method |
|----|--------|-------------------|
| **Real-Time Performance** | 96,000 sps target / 48,000 sps minimum / 192,000 sps stretch | FPS + processing time measurement |
| **Low Latency** | Minimize capture → process → GUI display end-to-end | 3-point latency measurement |
| **Correctness** | Displayed values ↔ actual acoustic events match | Comparison against WeiShi 1000 reference |
| **Measurement Accuracy** | Precise T1 onset/peak detection | Beat event accuracy verification |
| **Extensibility** | New graphs/filters addable without major changes to existing code | Module view-based impact analysis |

## Required Graphs

| Graph | Description |
|-------|-------------|
| Trace Display | Continuous real-time recording of rate deviation + amplitude (with smoothing) |
| Rate & Amplitude Stability (Vario) | Min/Max/Avg/σ statistics, acceptable range distinction |
| Multi-Position Sequence Display | Compare up to 10 position measurement results (including X·D summary values) |
| Beat-Noise Scope (Scope 1 & 2) | Individual beat noise waveform + Σ averaging |
| Beat Error Display & Diagnostic Trace | rate/amplitude/beat error values + trace graph |
| Long-Term Performance Graph | Long-term rate/amplitude/beat error changes |
| Escapement Analyzer & Marker-Line Display | A/C event markers + ms labels |
| Time-Frequency Spectrogram | Time-frequency energy distribution (color intensity = signal strength) |
| Waveform Comparison Display | Aligned continuous beat waveform comparison + timing markers |
| Scope Mode (Synchronized Sweep) | Oscilloscope-style fixed sweep window |
| Scope Function (F0/F1/F2/F3 Filter Views) | 4 filter processing views simultaneously |
